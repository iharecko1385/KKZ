## Vocabulary of English Terms

| Term | Definition/Explanation |
| :--- | :--- |
| Shell | A command-line program that interprets user inputs and acts as an interface to the operating system's kernel. |
| Command Line | The visual workspace where users type explicit text commands for the computer to execute. |
| CLI | Command Line Interface. A text-driven system UI used to run programs, manage files, and interact with the OS. |
| Prompt | The visual cue (like $) indicating that the shell is active and waiting for a new user command. |
| Command | An instruction passed to the system, triggering a specific built-in function or an external program. |
| Argument | The target data (such as a file path or specific text) that a command applies its action to. |
| Variable | A designated memory space with a specific name used to temporarily store text or numeric values. |
| Alias | A user-defined abbreviation or customized shortcut for a longer, more complex shell command. |
| Scripting | Combining multiple shell commands into a single executable file to automate repetitive tasks. |
| Built-in command | A command integrated natively inside the shell program itself, executing without external binary files. |
| External command | An independent executable file stored within the system directories (e.g., /bin or /usr/bin). |
| History | A shell function that logs previously typed commands, enabling fast search and re-execution. |
| Manual page (man) | The built-in documentation system that provides comprehensive guides on how to use specific commands. |

## Хід роботи

### Теоретичні відомості

* Командний інтерпретатор: Програмний комплекс, який зчитує текстові вказівки користувача, аналізує їхній синтаксис та передає на виконання ядру ОС або ж опрацьовує самостійно (вбудовані інструкції).
* Оболонка (Shell): Основний текстовий інтерфейс у середовищах UNIX/Linux. Найпопулярнішим варіантом оболонки є Bash (Bourne Again SHell).
* Команда: Рядок, що запускає певну дію. Це може бути виклик зовнішньої утиліти, системного скрипта, або вбудованого в оболонку інструмента.

### Відповіді на питання

1. Яку базову інформацію надає рядок запрошення (prompt)?
Цей рядок показує поточний контекст робочої сесії. Зазвичай він має вигляд user@host:~$ і вказує:
* Логін поточного користувача.
* Мережеве ім'я комп'ютера (хост).
* Директорію, в якій зараз знаходиться користувач (~ означає домашню папку).
* Рівень привілеїв ($ для звичайного доступу, # для прав адміністратора/root).

2. Для чого команді потрібні параметри та аргументи?
* Параметри (опції): Модифікують стандартну поведінку утиліти (зазвичай позначаються знаком дефіса, наприклад -a). Вони змінюють спосіб виконання команди.
* Аргументи: Вказують на цільовий об'єкт (наприклад, конкретний файл або шлях до папки), над яким команда повинна виконати дію.

3. Яке призначення команди `ls`, які параметри та аргументи вона може мати? Наведіть 3 приклади.
Утиліта ls використовується для відображення вмісту каталогів.
* ls — просто виводить список імен файлів та папок у поточній директорії.
* ls -la — виводить деталізований список усіх файлів (параметр -l), включаючи приховані (параметр -a).
* ls /etc — показує вміст конкретної директорії /etc (використання аргументу).

4. Яким чином можна використати історію команд, які переваги це надає?
Доступ до історії здійснюється через:
* Стрілки на клавіатурі (вгору/вниз) для перегортання попередніх запитів.
* Команду history, що друкує весь збережений журнал.
* Комбінацію Ctrl+R для швидкого текстового пошуку по історії.
Це суттєво заощаджує час на друкування довгих інструкцій та допомагає уникати помилок при їхньому повторному введенні.

5. Яке призначення команди `echo`?
Вона призначена для виведення вказаного тексту або значень конкретних змінних безпосередньо у вікно термінала.

6. Охарактеризуйте поняття змінної в оболонці Bash, які типи змінних вона підтримує?
Змінна — це іменована область пам'яті для тимчасового збереження інформації. В Bash існують:
* Локальні змінні: Працюють виключно в рамках того термінала, де були створені.
* Змінні оточення (глобальні): Передаються всім дочірнім процесам та іншим програмам, що запускаються з цієї оболонки.

7. Яке призначення команд `env`, `export` та `unset`?
* env — виводить список усіх активних глобальних змінних середовища.
* export — переводить локальну змінну в статус глобальної.
* unset — повністю видаляє змінну з пам'яті.

8. Які команди для отримання довідки по командам в терміналі ви знаєте?
* man <назва> — відкриває офіційний повний посібник.
* <команда> --help — виводить швидку довідку по ключах просто в термінал.
* info <назва> — детальна документація зі зручною навігацією по розділах.
* whatis <назва> — надає дуже короткий опис призначення утиліти (один рядок).

### Відповіді на контрольні запитання

1. Які типи команд існують в оболонці Bash?
В архітектурі Bash виділяють чотири категорії інструкцій:
* Внутрішні (Built-in): Команди, які реалізовані безпосередньо у вихідному коді самої оболонки (cd, pwd, echo).
* Зовнішні: Окремі скомпільовані програми, що лежать на диску системи (cat, ls, grep).
* Аліаси: Користувацькі скорочення для швидкого виклику інших команд.
* Функції: Скриптові blocks, що містять серію команд, згрупованих під однією назвою.

2. Що таке змінні оточення? Які вони бувають. Як їх можна переглянути в терміналі?
Змінні оточення — це системні параметри, які містять дані для конфігурації робочого середовища програм. 
* Вони поділяються на локальні (діють лише в поточній вкладці термінала) та глобальні (передаються всім програмам, які ви запускаєте).
* Для перегляду всіх глобальних змінних слід виконати команду env. Для виводу значення окремої змінної використовують echo $НАЗВА.

3. Опишіть змінну $PS1. Як в терміналі переглянути її вміст?
$PS1 (Prompt String 1) — це спеціальна змінна, яка форматує вигляд основного рядка запрошення до вводу. Вона визначає, чи будете ви бачити ім'я користувача, час або шлях до поточної папки ліворуч від курсору. Щоб побачити її синтаксис, введіть echo $PS1.

4. Як можна змінити значення змінної $PS1? Що при цьому відбудеться в рядку запрошенні в bash. Як змінити значення за замовчуванням?
* Тимчасова зміна: Команда PS1="Термінал> " одразу змінить вигляд запрошення, але налаштування зникне після закриття вкладки.
* Постійна зміна: Щоб зберегти новий вигляд назавжди, команду export PS1="..." треба прописати у конфігураційний файл ~/.bashrc.

5. Для чого використовують лапки в оболонці Bash?
Лапки потрібні для управління тим, як Bash розпізнає спеціальні символи та текст:
* Одинарні (`' '`): Суворе екранування. Увесь текст всередині сприймається буквально, змінні не розкриваються.
* Подвійні (`" "`): М'яке екранування. Текст сприймається буквально, але інтерпретатор розпізнає та підставляє значення змінних (знак $).
* Зворотні (`` ` ` ``): Використовуються для підстановки. Спочатку виконується команда всередині лапок, а її результат вставляється у зовнішню команду.

6. Для чого використовують інструкції керування, які їх види Ви знаєте?
Вони дозволяють об'єднувати команди у складні логічні ланцюжки:
* `;` — послідовне виконання команд одна за одною за будь-яких умов.
* `&&` (логічне І) — наступна команда запуститься тільки у разі безпомилкового виконання попередньої.
* `||` (логічне АБО) — наступна команда запуститься тільки якщо попередня завершилася з помилкою.

7. В чому різниця якщо в кінці рядку запрошення bash стоїть символ $ чи #?
Ці знаки візуалізують поточні права доступу в системі:
* `$` вказує на те, що сесія запущена від імені звичайного користувача з базовими правами.
* `#` є індикатором сеансу суперкористувача (root), який має абсолютний контроль над операційною системою.

8. Яке призначення команд whereis та locate? Яка між ними відмінність?
Команди створені для пошуку, проте мають різний механізм:
* whereis зосереджена виключно на системних компонентах: вона шукає шляхи до виконуваного файлу команди, її сирців та довідкових мануалів.
* locate — це інструмент глобального пошуку. Вона дозволяє миттєво знаходити будь-які файли на комп'ютері за допомогою попередньо проіндексованої бази даних системи (саме тому вона працює швидше, але може не помітити щойно створені об'єкти).
